# Fixed Point FIR Verification

## 1. Overview
In this project, we design a simple FIR (Finite Impulse Response) filter and study how it behaves in both software and hardware. The filter processes a stream of input numbers and produces an output by combining the current input with a small number of past inputs.

The project begins with a MATLAB model that describes how the filter should behave mathematically. This model is first written using floating-point arithmetic to verify correctness, and then rewritten using fixed-point arithmetic to closely match real hardware behavior.

Using the MATLAB fixed-point model, test vectors and reference outputs are generated. These reference results are then used to validate a Verilog hardware implementation of the same FIR filter. By comparing the Verilog output against the MATLAB reference, we ensure that the hardware implementation functions correctly across different input values and corner cases.


## 2. Directory Structure

| File | Description |
| :--- | :--- |
| `fir_gen_vectors.m` | **MATLAB Script.** Generates noisy input data, calculates the "Golden Reference" output, and exports `.txt` files. |
| `simple_fir.sv` | **RTL Design.** The SystemVerilog code describing the 8-tap filter hardware (Shift Register + Accumulator). |
| `tb_fir.sv` | **Testbench.** A simulation wrapper that reads `input_data.txt`, drives the RTL, and logs the output. |
| `input_data.txt` | **Stimulus.** Generated by MATLAB. Contains the noisy input integers. |
| `expected_out.txt` | **Reference.** Generated by MATLAB. Contains the expected filtered output integers. |

## 3. Filter Specification
* **Type:** 8-Tap Moving Average (Low Pass).
* **Math:** $y[n] = \frac{1}{8} \sum_{k=0}^{7} x[n-k]$
* **Input Format:** 8-bit Signed Integer (-128 to +127).
* **Output Format:** 8-bit Signed Integer.
* **Arithmetic:** Fixed-Point (Integer arithmetic). Division by 8 is implemented as an arithmetic right shift (`>>> 3`).

## 4. How to Run the Project

### Step 1: Generate Reference Vectors
1.  Open MATLAB or GNU Octave.
2.  Run the script `fir_gen_vectors.m`.
3.  **Result:** Two files (`input_data.txt` and `expected_out.txt`) are created in your working directory.
    * *Note:* The script uses `rng(1)` to ensure the random noise is identical for every run.

### Step 2: Run Hardware Simulation
- Ensure the generated `input_data.txt` is in the same folder as your SystemVerilog files.

**Cadence Xcelium**
I have used Texas A&M server to run the simulation. If you don't have access to universtiy server, you can use eda playground to simulate. 

- Run the following command in your terminal:
```bash
xrun -sv -access +rwc -gui simple_fir.sv tb_fir.sv
```

### Step 3: Verify Results
* Observe the simulation output log (e.g., Time: 550 | Output: 11).

* Compare these values with expected_out.txt generated by MATLAB.

* Success Criteria: The hardware output should match the MATLAB reference exactly.
